## Learnings & Corrections

- **[LEARN:env]** `DescTools::Winsorize` API changed at 0.99.60: signature became `Winsorize(x, val = quantile(x, probs = c(0.05, 0.95), na.rm = FALSE))` — the legacy `probs` / `na.rm` top-level arguments are removed. R port code that follows old vignettes (`Winsorize(x, probs = c(0.01, 0.99), na.rm = TRUE)`) fails with "unused arguments". Workaround: `Winsorize(x, val = stats::quantile(x, probs = c(0.01, 0.99), na.rm = TRUE))`. Affects outlier-spec §6.3 / §6.5 R porting and any future Winsorize call. Fix is local; spec intent preserved.
- **[LEARN:env]** `install.packages()` default 60-second timeout is too aggressive for CRAN binary downloads on slow links (4 of 16 packages timed out: stringi, stringmagic, performance, ggplot2). For one-shot setup scripts targeting fork users / new machines, set `options(timeout = 600)` before `install.packages()`. `_setup_packages.R` should adopt this pattern.
- **[LEARN:r]** R `cat(..., file = con)` with multiple `cat()` calls on the same opened connection is fragile — observed "invalid connection" mid-write on a fresh `file(open = "wt", encoding = "UTF-8")` even when handlers are correctly registered. Replace with `writeLines(c(line1, line2, ...), con = path)` single-call pattern for short text logs. Robust + simpler.
- **[LEARN:methods]** First-stage take-up signal already visible in raw `actual_subsidy` (#84): treated post-2020 median = 1,292,240 KRW (≈ SFFP flat 1,200,000 KRW); control post-2020 median = 3,512,265 KRW (area-proportional, larger farms get more); treated pre-2020 median = 328,660 KRW (legacy programs only); control pre-2020 median = 2,086,855 KRW. Discontinuity at the 0.5 ha cutoff visible without any regression. Confirms the SFFP discontinuity is sharp enough for the DiD-RD design to identify a meaningful first stage.
